5 Source File Character Encoding

The Cminor compiler assumes that source files have implicit UTF-8 encoding. UTF-8 byte order mark EF BB BF can be used in the start of a source file, but it's not required.

All the string literals contained in source files are converted from UTF-8 to UTF-32 representation internally. The Cminor string and character types have internal UTF-32 encoding.

The output encoding of strings and characters is UTF-8.
